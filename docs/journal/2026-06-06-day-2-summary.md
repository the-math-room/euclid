# 2026-06-06: Day 2 Summary — Headless Package Architecture and New Construction Tools

## Development Focus

Day 2 focused on decoupling the monolithic layout into distinct packages, refining core geometry math, establishing a curriculum and assessment layer, and integrating new construction tools (parallel lines, perpendicular lines, and midpoints).

The architecture is centered on a headless model: `@euclid/geometry` is the core package that works without rendering, DOM, or browser contexts. The Web Studio acts as an interpreter of this geometry state.

---

## Core Geometry Refinement

We addressed several functional and type-safety gaps in the `@euclid/geometry` package:

- **Branded Coordinates**: Introduced compile-time branded types (`WorldPoint` and `ScenePoint`) on top of `Point2`. This prevents accidental mixing of world-space math coordinates and screen-space pixels at the compile level.
- **Direct Manipulation**: Implemented shape dragging. Dragging a line or circle translates its defining free point dependencies in world space, triggering automatic recalculation of the evaluation graph.
- **Viewport Controls**: Added smooth keyboard-based viewport controls (pan, zoom, rotate via arrow keys and brackets) using custom key-repeat handling to bypass OS key-repeat delays.

---

## Headless Package Extraction

To isolate responsibilities, we extracted three package layers from the application core:

- **`@euclid/activity`**: Manages activity-level permissions (e.g., allowed tools, locked constructions, delete permissions, and drag restrictions). Exports `activityTools` and `isActivityTool` for tool validation.
- **`@euclid/assessment`**: Contains semantic assessment predicates (`requiresMeaning`, `requiresDependency`, `requiresPointOnLine`, etc.). This compares the structural relationships in the construction dependency graph (`sameExpression`) instead of doing pixel or label checks, making goal evaluation robust to renaming and alternative construction routes.
- **`@euclid/lesson`**: Composes a starter document, an activity policy, and serializable assessment goals into a portable `.lesson.json` file format.

---

## Web Studio & Lesson Player Integration

We integrated the extracted packages into the React application as a dynamic lesson player:

- **Goal Resolution**: Built a dynamic ID resolver to resolve mismatches between static IDs in lesson templates (e.g., `"line-ab"`) and dynamic IDs generated by user drawing (e.g., `"line-a-b"`). This resolver evaluates expressions structurally and maps user IDs to lesson goals. The logic was extracted to `packages/assessment/src/goalResolution.ts` to keep the app layer decoupled.
- **Lesson Persistence**: Integrated state management to persist and reset learner programs keyed by stable `EuclidLesson.id` strings.
- **Camera Pivot Fix**: Solved a camera rotation pivot bug. Panning now updates the camera's world-space center directly instead of accumulating a screen delta, guaranteeing that camera rotation pivots around the center of the current viewport.
- **Rendering Purification**: Removed module-level caches from the label layout solver in `@euclid/rendering` to ensure rendering remains side-effect-free.

---

## New Construction Tools & Tech Debt Resolution

We added three new construction primitives to the core stack:

- **Parallel Lines**: Renders a line through a witness point parallel to a reference line.
- **Perpendicular Lines**: Renders a line through a witness point perpendicular `(-dy, dx)` to a reference line.
- **Midpoints**: Computes the arithmetic mean of two parent point coordinates. Midpoint comparisons in assessment are order-independent.
- **Declarative Gesture Policies**: Resolved pointer targeting bugs (e.g. duplicate point aliases when clicking near intersections) by introducing `ConstructionToolGesturePolicy` configuration tables in `apps/web/src/construction/tools.ts`. The gesture controller looks up priority lists (e.g., preferring existing points, lines, intersections, or empty space) per-tool rather than using nested conditional branches.
