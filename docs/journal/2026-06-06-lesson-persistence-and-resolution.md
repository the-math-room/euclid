# 2026-06-06: Lesson Persistence, Step Feedback, and Dynamic ID Resolution

## Summary

Successfully integrated `@euclid/lesson` parsing, `@euclid/activity` policy enforcement, and `@euclid/assessment` goal evaluation directly into the web application shell, transforming Euclid into a curriculum activity player.

## What Changed

- **Granular Step Feedback**:
  - Evolved `AssessmentGoal` variants to include an optional `description` string.
  - Flattened the goals arrays in `lessons.ts` to provide step-by-step objectives.
  - Toggled objectives in the UI panel with checkmarks as students complete specific steps.

- **Dynamic ID Resolution**:
  - Implemented `resolveGoalMapping` in a new utility `assessmentResolver.ts`.
  - Resolved mismatches between static IDs in lesson definitions (e.g. `"line-ab"`) and dynamic IDs generated by user drawing (e.g. `"line-a-b"`).
  - Recursively traverses goals (including nested `all` and `any` goals) to match expressions structurally and substitute user IDs.

- **Activity State Persistence & Reset**:
  - Lifted lesson program state to the parent `App` component as a map from lesson index to program, maintaining progress when switching lessons.
  - Added a header toolbar dropdown to swap between active lessons.
  - Integrated a "Reset" button that clears the active lesson's state and uses a version salt key to force-remount the `WorkspaceContainer`.

## Strategic Meaning

This completes the product-facing goal from the strategic plan: letting the web app interpret a `EuclidLesson` as its starting point. We kept React as an interpreter of evaluated geometry rather than its owner, preserving the clean denotational boundaries of the SDK packages.
